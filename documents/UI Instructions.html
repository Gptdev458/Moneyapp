Instructions for AI Agent: HTML to React Native TSX Conversion

Objective:

Your task is to convert the provided HTML screen designs (found below these instructions) into fully functional, well-structured React Native components using TypeScript (TSX). The goal is to replicate the visual appearance, layout, and basic structure of each screen as accurately as possible.

Important: Each screen's HTML content is typically found within a <div class="[screen-name]-phone-frame">. Focus on converting the content *within* this frame for each respective screen.

Core Technologies & Libraries:

  • React Native: Use standard React Native components (View, Text, TouchableOpacity, ScrollView, FlatList, TextInput, Image, SafeAreaView, StatusBar).
  • TypeScript: All components should be .tsx files with strong typing for props and state.
  • Styling: Use React Native's StyleSheet.create({}) for all styles.
  • SVG Icons: Use react-native-svg for converting the inline SVGs from the HTML into reusable icon components.

General Principles:

  1. Componentization:
    • Break down each screen into logical, reusable functional components (e.g., ScreenHeader, BalanceCard, BudgetListItem, TransactionRow, BottomNavigationBar).
    • Create components for shared UI elements if they appear on multiple screens (e.g., the bottom navigation bar).
  2. TypeScript & Props:
    • Define clear interface for props for every component.
    • Pass all dynamic data (text, numbers, colors, icon names, event handlers) to components via props. The static content in the HTML will serve as the basis for these props.
  3. Styling with StyleSheet:
    • Translate all Tailwind CSS classes and any inline styles from the HTML into React Native StyleSheet objects.
    • Color Mapping: The HTML uses a dark theme. Map Tailwind color classes (e.g., bg-gray-900, text-blue-400) to their corresponding hex color codes. Refer to the "Color Palette Reference" section below.
    • Layout (Flexbox): Convert Tailwind flexbox utilities (flex, flex-row, justify-between, items-center) to React Native's flexbox style properties.
    • Sizing & Spacing:
      • Map Tailwind spacing (p-, m-, space-x-) and sizing (h-, w-) utilities to React Native styles. Assume 1 Tailwind unit = 4px (e.g., px-6 becomes paddingHorizontal: 24, h-10 becomes height: 40).
      • For space-x-N or space-y-N, apply margins to the child elements (except the first for space-x or last for space-y in some cases, or use gap if parent is flex container).
    • Borders & Rounded Corners: Convert classes like border-b, rounded-lg, rounded-full to React Native border and borderRadius styles.
    • Shadows: For shadow-lg, use React Native's elevation (Android) and iOS shadow properties (shadowColor, shadowOffset, shadowOpacity, shadowRadius). A common dark shadow might be: shadowColor: '#000', shadowOffset: { width: 0, height: 2 }, shadowOpacity: 0.25, shadowRadius: 3.84, elevation: 5.
  4. SVG Icons:
    • Each unique inline SVG from the HTML should be converted into a separate React Native functional component using react-native-svg.
    • These icon components should accept optional size (defaulting to ~24) and color (defaulting to a common text color like white) props.
    • Create a central IconComponents.tsx file to house these, and an IconCollection object for easy named access.
  5. Text Content: Use the exact textual copy from the HTML.
  6. Interactivity:
    • HTML <button> tags (or divs styled as buttons) should become React Native <TouchableOpacity> or <Pressable> components.
    • Input fields (<input>, <textarea>, <select>) should be implemented using <TextInput>. For <select>, you might need a custom component or a library, but a styled TextInput with a dropdown arrow icon can be a starting point if a full picker is too complex for this stage.
    • Assign placeholder onPress handlers (e.g., const handlePress = () => console.log('Element Pressed');) to all interactive elements.
  7. Data Structures:
    • For lists of items (transactions, budgets, accounts), define TypeScript interfaces for the data objects (e.g., TransactionData, BudgetItemData).
    • The main screen component will typically hold an array of these data objects (initially as static/dummy data) and use .map() or <FlatList> to render child components for each item.
  8. Scrollability: Ensure that content exceeding the screen height is scrollable using <ScrollView> or <FlatList>. The phone-frame divs in the HTML often have overflow-y-auto on list containers.
